# 📝 v3 内容改进 — 2026-06-19

## 背景

前几篇文章（Brainstorming、Feedback、Exit Interview v1）反馈：
- "内容太少太浅"
- "需要增加难度和深度"
- "排版展示效果不好，列表序号字体大、正文被挤"

## 本次改动

### 1. 内容深度升级
- **P1 表达深度分析**：每个表达拆三层——字面含义 → 语气潜台词/文化差异 → 进阶用法/扩展
  - 示例：分析 "move on" 时不只是翻译，还说明跟 "leave" 的区别、"move on to a new opportunity" 比 "I'm leaving" 更安全的潜台词
- **三大尾栏板块**：词汇(3-5个+例句)、句型(2-3个+场景说明)、练习(场景化留言任务)
- 字数从 500-800 逐步提升到 1200-1500

### 2. 开头画面感
- 每篇文章开头从 3 句话扩充到 4-6 句话
- 增加时间/地点细节（"早上 9:25"）、动作描写（"把咖啡端到工位"）、心理活动（"你心里咯噔一下"）
- 目标：让读者 3 秒内产生"这说的就是我"的代入感

### 3. 排版修复（关键）
- 列表渲染完全重写：弃用 `<ol>/<ul>/<li>`，改用 `display:flex` div 方案
- 序号/圆点用 CSS 精确控制尺寸，WeChat 手机端无法覆盖
- 正文用 `flex:1` 填满剩余空间，不会被挤成窄条

### 4. 场景库扩展
- 新增 4 个轻松话题：闲聊、入职、团建、出差
- 按 3:1 配比穿插实用场景和轻松内容

## 效果
- Morning Standup Meeting（v3 第一版）：
  - 开头：有现场感的 5 句话场景描写
  - "I was working on" 拆解了三层
  - 段落短小，手机阅读舒适
  - 尾栏词汇/句型/练习齐全
  - 列表全部 flex 渲染

## 待观察
- 读者对三大尾栏的反应（互动率是否提升）
- 轻松话题的阅读量是否高于实用话题
- flex 列表在真实手机上的最终显示效果
